# Contributing

Thanks for stopping by. Contributions of all sizes are welcome — a typo fix is just as appreciated as a new agent.

## Before You Start

- Check existing issues and PRs to avoid duplicating effort
- For big changes, open an issue first to discuss the direction

## How to Contribute

### Reporting Bugs

Open an issue using the **Bug Report** template. Include enough detail to reproduce the problem.

### Suggesting Features

Open an issue using the **Feature Request** template. Explain the problem you're solving, not just the solution.

### Submitting Code

1. Fork the repo
2. Create a branch: `git checkout -b your-feature-name`
3. Make your changes, keeping commits small and focused
4. Add or update tests if relevant
5. Open a pull request against `main`

## Code Style

- Follow the conventions already in the repo
- Keep it simple — complexity should earn its place
- One thing per repo, one responsibility per module

## Pull Request Checklist

- [ ] Changes are focused and minimal
- [ ] Code is self-explanatory or commented where it isn't
- [ ] Tests pass (if applicable)
- [ ] README updated if behavior changed

## Questions?

Open a discussion or drop a comment on a relevant issue. We're friendly.
